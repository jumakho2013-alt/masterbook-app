# 15 concrete UX changes for MasterBook — ordered by impact/effort

## P0 — biggest unlock per hour of work

### 1. Add a "Try with demo data" path on Welcome — defer onboarding choices
**Where:** `app/(auth)/welcome.tsx`, new helper `src/data/demo-seed.ts`, `useAuthStore`  
**What:** Welcome currently has one button "Начать". Add a second, equally-prominent button "Посмотреть как работает" that seeds 3 demo clients, 5 services for the user's likely profession (use Apple/Android system locale + first-name heuristic, or ask one question), and 4 appointments (yesterday completed, today in-progress, tomorrow, next week). Mark them as `isDemo: true` so they can be wiped with one tap. Profession + name can be asked later from a non-blocking banner on the Today screen.  
**Why:** The current flow is welcome → profession → specialization → services-setup (with name input) → tabs. 4 screens before the user sees a single screen of the actual product. CIS users are skeptical and bounce. Demo data answers "что это вообще такое" in 5 seconds.  
**Impact:** Activation +15–25%, especially on installs from store browsing (not direct referral).

### 2. Collapse onboarding from 4 steps to 2 — kill the standalone "Ваше имя" field
**Where:** `app/(auth)/services-setup.tsx`, `app/(auth)/specialization.tsx`, `app/(auth)/profession.tsx`  
**What:** Merge specialization into profession (a sectioned list with chips, not a separate screen — most categories have 2–4 specializations). Move "Ваше имя" out of services-setup entirely — either pre-fill from the Apple/email auth profile, or ask on first save of a client/appointment ("Как клиенту представить вас?"). The services-setup screen is the only one that justifies its own page — that's where the user gets the "ah, оно само настроилось" moment.  
**Why:** Each extra screen = ~15–20% drop. Profession→specialization split exists for code symmetry, not UX value. Name input on a service screen is cognitive friction with no payoff (it's only used in PDF/exports, which most users never touch).  
**Impact:** Onboarding completion +10–15%; less keyboard time on first run.

### 3. Make the FAB context-aware on "Сегодня" — primary action is "+ запись для X" when there's an obvious client
**Where:** `app/(tabs)/index.tsx`  
**What:** When a "Сейчас идёт" card is shown OR there's a completed visit in the last hour with no follow-up, the FAB labels itself "+ Повторить" or "+ Записать на след. раз" and pre-fills the new-appointment flow with that client. Long-press still opens the full new-appointment flow. Visually: small client avatar fragment in the FAB.  
**Why:** Solo masters' #1 task right after a visit is rebooking. Currently they tap +, then search the client, then pick a service, then a date. 4 screens → 1.  
**Impact:** Retention — rebookings drive LTV.

### 4. Replace 3-tab finance period selector with "This month" default + swipeable view
**Where:** `app/(tabs)/finances.tsx`  
**What:** Default to "Месяц" (already is, good). Hide "День/Неделя/Месяц" tabs behind a small period chip in the header; swap them for **horizontal swipe between months** (current month / previous month / two months ago) with a date pill at top. Add a one-line plain-Russian summary at the very top: *"В июне заработали 87 400 ₽, это на 12% больше мая."* No icons, no charts above the fold for first paint.  
**Why:** "Финансы" looks like an accountant's spreadsheet today (Доход/Расход/Чистыми + chart + top clients + popular service + transactions). For a manicurist who hates math, the first thing they need is **one sentence in human language**. Charts and breakdowns belong below the fold.  
**Impact:** Tab visit duration +30%, perceived value of the tab up significantly.

## P1 — high-value, medium-effort

### 5. Add a global search via pull-down or top-bar — payable surface for "поиск клиента/записи"
**Where:** New `src/components/GlobalSearchSheet.tsx`, hook into `app/(tabs)/_layout.tsx` for swipe-down on any tab  
**What:** Pull-down anywhere reveals a modal sheet with one input. Searches across clients (name, phone, notes, preferences), upcoming appointments (by client name), services (by name), past visits (by service name). Each result row shows the entity icon and tappable shortcut (Call / Message / Open).  
**Why:** Today, finding "когда у меня была Маша последний раз" requires Clients → search → tap → scroll. 4 taps. Global search is 1 swipe + 1 type.  
**Impact:** Power-user retention; reduces "где же эта запись" moments that erode trust.

### 6. Quick-add appointment via long-press on a calendar day
**Where:** `app/(tabs)/calendar.tsx`  
**What:** Long-press a day in week or month strip → opens new-appointment flow with date pre-filled. Long-press a time-slot row inside the day list → pre-fills date + time. Subtle haptic + visual ripple.  
**Why:** The standard flow goes through the FAB which forces step 1 = client. But masters often think "у меня в четверг 14:00 окошко, кого бы вписать" — they want to start from a time. Long-press lets them.  
**Impact:** Calendar feels like a tool, not a viewer. Pro masters love it; novices don't notice.

### 7. Add a "Завершить и записать на след. визит" combo action on the appointment detail
**Where:** `app/appointment/[id].tsx`  
**What:** Currently completing a visit and creating the next one are separate flows. Add a single button: marks current as completed → if client has a preferred interval (default = service interval), opens new-appointment pre-filled with same client/service/+N weeks. Save in one tap, or adjust.  
**Why:** This is the *actual job* — most services have a natural rhythm (manicure 3w, hair 4–6w, lashes 3w). The app should make repeat scheduling the default, not a 6-step path.  
**Impact:** Per-client appointment density goes up — exactly what a CRM should drive.

### 8. Day-of summary push at 8:00 + evening "no-show / уточнить оплату" nudge
**Where:** `src/lib/notifications.ts` + settings UI in `app/settings/`  
**What:** Today only sends one local notification 1h before each appointment — that's the right floor. Add two more:
- **Morning summary (08:00 default)**: "Сегодня 3 записи, прогноз 7 800 ₽. Первая в 10:30 — Анна, маникюр."
- **Evening review (20:00)**: only if there are appointments today flagged completed without payment recorded, OR no-show candidates (start time passed > 30 min, still `scheduled`) — "Подтвердите визиты дня". Defaults on, both togglable. Don't add weekly summaries — too much.  
**Why:** A CRM's job is to *show up at the right moment*. 1h-before is reactive; 8:00 is the "boot up the day" trigger; 20:00 closes the loop on payments (which is what eventually drives the user to enter finance data themselves).  
**Impact:** Daily-active sessions +20–30%; finance data quality goes up because the app asks at the right moment.

### 9. Reachable controls — move calendar's "Сегодня" + view-mode toggle to a bottom action band on Android
**Where:** `app/(tabs)/calendar.tsx`, `app/(tabs)/clients.tsx` (sort chips), `app/(tabs)/finances.tsx` (period tabs)  
**What:** On Android phones held one-handed, ~6.5" screens, the top-right "Сегодня" button is unreachable with the thumb. Move "Сегодня" + view-mode toggle to a floating pill at the bottom-right of the calendar tab (separate from the FAB which lives bottom-center-right). On Clients, the horizontal sort scroll lives at the top — fine because it's already big — but the search bar should sit *above* sort chips (currently does, good) and chips should be sticky on scroll. On Finances, the period chips at the top of the FlatList should also sticky-pin (use `stickyHeaderIndices={[0]}`).  
**Why:** AUDIT marked accessibility wins for contrast/labels, but ergonomics on one-handed Android were not addressed. Liquid Glass FABs and tab bar are already at thumb-zone — secondary controls should match.  
**Impact:** Subjective "приятно пользоваться" — drives ratings on Play.

## P2 — quick wins, low effort

### 10. Empty-state CTAs that *do* something, not just describe
**Where:** `app/(tabs)/index.tsx`, `app/(tabs)/clients.tsx`, `app/(tabs)/finances.tsx`, `app/(tabs)/calendar.tsx`  
**What:** Replace passive empty states ("Нет записей" + subtitle) with action-shaped ones:
- Today (no appts): big card "Сегодня выходной?" with two buttons — "Записать клиента" and "Закрыть день". 
- Clients (0): "Импортировать из контактов" (uses `expo-contacts`) + "Добавить вручную". The contacts import is a 30-line addition that unlocks 5–50 clients in 10 seconds — single biggest activation lever.
- Finances (0): "Все доходы уже считаются автоматически по завершённым визитам. Добавьте первую запись."  
**Why:** "Sub-empty" states (icon + grey text) are the dead air of mobile apps. Every blank canvas is a chance to make the user act.  
**Impact:** Clients-import alone — measurable, probably the single most important change in this list for activation. Worth promoting to P0 if the team has 1 day to do it right.

### 11. Microcopy pass — make it sound like a co-worker, not a form
**Where:** Across all files; consolidate into `src/lib/copy.ts` for future i18n  
**What:** Specific rewrites:
- "Чем вы занимаетесь?" → "Кто вы по профессии?" (более естественно)
- "Готово" (services-setup) → "Поехали"
- "Добавлю позже" → "Пропустить пока"  
- "Транзакции" (finances) → "История доходов"
- "Нет клиентов" → "Тут будут ваши клиенты"
- "Никого не нашли" → "По запросу ничего"
- "Введите имя" (validation error) → "Имя обязательно — без него я не сохраню"
- "Davно не были" (sleeping clients block title) → "Можно напомнить о себе"
- Confirm screen labels (Клиент / Услуга / Дата / Время) — fine as-is, that's correct register
- Profession screen title `'Чем вы\nзанимаетесь?'` — drop the manual line break, let it wrap  
**Why:** Form-y register signals "это бухгалтерия для меня" — exactly the wrong vibe for the target user.  
**Impact:** Hard to measure but compounds — affects every screen.

### 12. Dark mode tuning for OLED — pure-black background + softer surface ladder
**Where:** `src/theme/colors.ts`  
**What:** `darkColors.background: '#13131A'` is dark-grey, not OLED-black. On OLED phones this wastes battery and looks like fake-dark. Change to:
- `background: '#000000'` (true OLED black)
- `surface: '#15151E'` (slight lift)
- `surfaceElevated: '#1F1F2A'` (sits on surface)
- `surfaceGlass: 'rgba(20,20,28,0.78)'`

Also: `primary: '#9B8AFB'` on `#000` background is ~6.3:1 contrast — fine for text, but the soft tint `rgba(155,138,251,0.15)` over pure black looks muddy. Bump to 0.18. Test "Сейчас идёт" indigo card on OLED — current `shadowColor: '#7C5DFA'` shadow is invisible on black; switch shadow on dark theme to a coloured ring or remove.  
**Why:** OLED Android phones (Pixel, Samsung) are the majority of CIS premium devices. True black is also a brand signal — "это серьёзное приложение".  
**Impact:** Reviews mention dark mode; battery savings real but invisible to user.

### 13. Make the client card's single most important action — call — a one-tap from the list, not from detail
**Where:** `src/components/ClientRow.tsx`  
**What:** Add a small phone icon button on the right of each `ClientRow` (replacing or alongside the chevron). One tap = `Linking.openURL('tel:...')`. Long-press the row still opens the full client detail. Use `accessibilityActions` to expose this to VoiceOver.  
**Why:** Detail screen has 4 contact action buttons (Phone/WhatsApp/Telegram/Plus) — beautiful but two taps from list. Calling a client to confirm is a daily action; saving 1 tap × 10/day matters.  
**Impact:** Daily action friction down; WhatsApp/Telegram remain one tap inside detail (acceptable).

### 14. Calendar default to **week view, not auto**, with a "scrollable timeline" mode unlocked from a long-press
**Where:** `app/(tabs)/calendar.tsx`  
**What:** Empirically, solo masters live in 7-day horizons. Week view is already default — keep. But the *secondary* mode (toggled by the LayoutGrid icon) should be a **vertical day timeline** (08:00 → 22:00 with appointment blocks proportional to duration), NOT a month grid. Month is for tax/reporting, used 1×/month. Move month grid to a small "view all" sheet from the calendar header chevron.  
**Why:** The current week ↔ month toggle conflates two unrelated jobs (overview vs. day-of). Day-timeline view is what masters tap *during* a workday to answer "когда следующий клиент".  
**Impact:** Calendar tab session length up; perceived usefulness up.

### 15. Accessibility — large-text mode + dynamic type respect
**Where:** Theme `src/theme/typography.ts` (not read but inferred); every `typo.h*`/`typo.body` consumer  
**What:** Audit shows `accessibilityLabel`/`accessibilityRole` are present (good). Missing: `allowFontScaling` is at React Native default, which is `true` — but fixed-height containers (date chips 56×?, time slots minWidth 84, FAB 56×56) will clip text at 200%+ system font size. Add a `useDynamicType` hook that returns `PixelRatio.getFontScale()` and either:
- Caps font scaling at 1.3 on critical pill components (`maxFontSizeMultiplier={1.3}` prop)
- Or makes those pills `minHeight: 44 * fontScale`

Also missing: high-contrast mode detection (`AccessibilityInfo.isHighTextContrastEnabled()` on Android). When on, swap `textTertiary` to `textSecondary` and bump border opacity to `0.16`.  
**Why:** Visually impaired users zoom system font; current chips will truncate or break layout. Motor-impaired users benefit from larger hit targets — already mostly OK (44pt with hitSlop), but the FAB at 56×56 close to the bottom tab bar may be too close to neighbour controls on Android tablets.  
**Impact:** Inclusivity; also Google Play accessibility scanner score for store ranking.
