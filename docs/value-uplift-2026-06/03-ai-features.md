# AI-Powered Automations для MasterBook — стратегия и дизайн

## Философия

AI в MasterBook должен экономить мастеру **5+ часов в неделю** на коммуникации, аналитике и рутине. Главный принцип — **AI как тихий ассистент, а не назойливый помощник**. Каждая AI-фича должна отвечать на вопрос: "что бы сделал хороший секретарь, если бы я ему делегировал это?".

Второй принцип — **profession-agnostic**. Тренер по теннису, репетитор по математике, фотограф, ветеринар, автомеханик, юрист — у всех одни и те же задачи: напомнить, не потерять клиента, не пропустить запись, посчитать налог.

## 1. Auto-drafted reminders (Авто-напоминания клиентам)

**Триггер:** за 24 часа до записи (настраивается).
**Входные данные:** имя клиента, услуга, время, история визитов (кол-во, последняя дата), preferences-заметки, стиль прошлых сообщений мастера (если согласился обучить), время суток.
**Выход:** черновик сообщения в выбранном тоне (формальный / дружеский / нейтральный / с эмодзи). Мастер свайпит "отправить" или редактирует.
**UX:** карточка в Inbox-секции главного экрана: "3 напоминания готовы. Просмотреть?". Один тап — пакетная отправка через WhatsApp/Telegram/SMS deep-link.
**Реализация:** **template + variables + tone-selector**, НЕ обучение на личных сообщениях (privacy + сложность). Tone задаётся один раз в onboarding ("Как ты обычно пишешь клиентам? Покажи пример"). На основе примера GPT-4o-mini генерит 5 шаблонов под разные ситуации (первый визит / постоянный / VIP / возврат после паузы).
**Модель:** GPT-4o-mini ($0.15/1M input). 1 мастер × 60 сообщений/мес × ~200 токенов = ~12k токенов = **0.3 ₽/мес**.
**Профессия-агностик:** да. Репетитор: "Маша, не забудь — завтра в 16:00 разбираем тригонометрию". Автомеханик: "Сергей, твоя машина готова, заберёшь в 17:00?".
**Priority: MUST. Complexity: S. Tier: PRO.**

## 2. No-show prediction (Риск неявки)

**Триггер:** при создании записи + за 24ч до записи.
**Сигналы:** % прошлых неявок, кол-во переносов, время с момента бронирования (записи >14 дней вперёд риск выше), новый клиент vs постоянник, день недели, время суток, был ли депозит, ответил ли на reminder.
**Выход:** badge на записи: "🟡 риск 35%" + рекомендация ("попроси предоплату 500 ₽ через СБП").
**UX:** маленький бейдж на appointment card. Тап → объяснение какие факторы повлияли (transparency = trust).
**Реализация:** **локальная** логистическая регрессия или простой rule-based scoring (5-10 правил с весами). Никакого LLM — для этого не нужен. Считается on-device.
**Модель:** on-device.
**Стоимость:** **0 ₽**.
**Профессия-агностик:** да, универсально.
**Priority: MUST. Complexity: M. Tier: PRO.**

## 3. Service duration auto-suggest

**Триггер:** при создании записи (после выбора клиента и услуги).
**Входные данные:** медиана длительности этой услуги у этого клиента за последние 5 раз; fallback — медиана по всем клиентам.
**Выход:** "обычно занимает 95 мин (твоё среднее — 75)". Один тап применить.
**UX:** inline-подсказка под полем длительности.
**Реализация:** **on-device**, простой group-by + median. Никакого ML.
**Стоимость:** **0 ₽**.
**Профессия-агностик:** идеально. Репетитор: "обычно занятие с Машей идёт 90 мин а не 60". Массажист, груминг собак, юрист-консультант — везде работает.
**Priority: MUST. Complexity: S. Tier: Free.**

## 4. Smart rescheduling (Перенос записи)

**Триггер:** клиент/мастер инициирует перенос.
**Входные данные:** свободные слоты на 7 дней вперёд, предпочтения клиента по времени (исторический паттерн — обычно ходит вечером в будни), длительность услуги, паузы мастера.
**Выход:** 3 рекомендованных слота с reasoning ("Анна обычно ходит вечером — вт 18:00, чт 19:00, пт 17:30").
**UX:** bottom-sheet "Предложить клиенту" с 3 кнопками-слотами; тап → готовое сообщение со слотами.
**Реализация:** **on-device**. Простой ranking: совпадение с паттерном клиента (вес 50%), близость к отменённому времени (30%), не край рабочего дня (20%). Погода — overkill, выкидываем.
**Стоимость:** **0 ₽**.
**Priority: NICE. Complexity: M. Tier: PRO.**

## 5. Receipt-to-expense (Чек → расход)

**Триггер:** "+ расход" → "сфотографировать чек".
**Входные данные:** фото чека.
**Выход:** распознанные поля — сумма, дата, продавец, категория (расходники / транспорт / аренда / реклама / прочее).
**UX:** превью с распознанными полями, мастер правит и сохраняет.
**Реализация:** GPT-4o vision (умеет русские чеки, ОФД QR, ИНН). Альтернатива — Yandex Vision OCR (дешевле, русский лучше) + локальная категоризация по словарю продавцов.
**Модель:** GPT-4o vision. ~0.5 ₽ за чек × 20 чеков/мес = **10 ₽/мес**.
**Профессия-агностик:** да, расходы у всех.
**Priority: NICE. Complexity: M. Tier: PRO.**

## 6. Income forecast (Прогноз дохода)

**Триггер:** dashboard.
**Входные данные:** подтверждённые записи до конца месяца × средний чек + историческое заполнение оставшихся дней (медиана за 3 предыдущих месяца того же месяца недели).
**Выход:** "До конца июня: ~85,000 ₽ (±12k). Сейчас забронировано 62%, обычно дозаписываешь ещё 28%."
**UX:** виджет на главном экране, доступен в финансах глубже.
**Реализация:** **on-device**, простая арифметика + доверительный интервал по std исторических данных.
**Стоимость:** **0 ₽**.
**Профессия-агностик:** да.
**Priority: MUST. Complexity: S. Tier: Free (базовый), точный с reasoning — PRO.**

## 7. Idle-slot suggestions (Заполнить окна)

**Триггер:** проактивно вечером в воскресенье; manual через AI-assistant.
**Входные данные:** свободные слоты на неделю, "спящие" клиенты (категория из feature #9), их предпочтения по времени, last-visit gap.
**Выход:** "Вт 14:00-16:00 свободно. Напиши: Лене (не была 67 дней, обычно вторник), Ольге (не была 52 дня), Кате (новый клиент, оставила контакт)". К каждому — draft-сообщение.
**UX:** карточка в Inbox: "💡 3 идеи как заполнить окна на неделе". Никаких push, только при открытии app.
**Реализация:** клиентский ranking on-device, drafts через GPT-4o-mini.
**Стоимость:** ~**2 ₽/мес** на drafts.
**Профессия-агностик:** да. Особенно ценно для репетиторов, тренеров, фотографов.
**Priority: MUST (KILLER FEATURE — это прямой доход). Complexity: M. Tier: PRO.**

## 8. Client churn alert (Клиент уходит)

**Триггер:** ежедневный фоновый скан (utilities task).
**Входные данные:** для каждого клиента — средний gap между визитами, текущий gap, std.
**Выход:** "Лена не была 11 недель — обычно ходит каждые 6. Сообщение готово." (Поведенческий outlier по личному паттерну клиента, не глобальный.)
**UX:** badge в списке клиентов + ноут в Inbox раз в неделю. Не чаще!
**Реализация:** **on-device**. (current_gap > avg_gap + 1.5 × std) AND (visits ≥ 3).
**Стоимость:** **0 ₽** (drafts опционально, ~1 ₽/мес).
**Профессия-агностик:** да. Особенно силен для регулярных услуг — массаж, барбер, репетитор.
**Priority: MUST. Complexity: S. Tier: PRO.**

**Стоимость:** **0 ₽**.

## 10. Voice-to-appointment (Голос → запись)

**Триггер:** микрофон-кнопка в FAB или Siri Shortcut / Assistant Action.
**Входные данные:** аудио "запиши Аню на четверг в три на маникюр с дизайном".
**Выход:** распарсенная запись — клиент (нечёткий matching по существующей базе), дата, время, услуга, длительность из feature #3.
**UX:** preview-screen, мастер подтверждает.
**Реализация:** Whisper API ($0.006/мин) → GPT-4o-mini structured output → fuzzy match к существующим клиентам.
**Модель:** Whisper + GPT-4o-mini. ~1 ₽ за запись × 30/мес = **30 ₽/мес**.
**Профессия-агностик:** да, и особенно мощно для мастеров "руки заняты" (массаж, парикмахер, ветеринар, автомеханик).
**Priority: NICE (wow-factor, демо-продающий). Complexity: M. Tier: ELITE.**

## 11. Photo analysis for portfolio

**Триггер:** мастер прикрепил фото к записи "до/после".
**Входные данные:** фото.
**Выход:** "качество ✅ — годится для портфолио" или "размыто/плохой свет — переснять?".
**UX:** иконка-badge на фото в галерее.
**Реализация:** on-device CoreML / Vision API (blur detection, sharpness). LLM не нужен.
**Стоимость:** **0 ₽**.
**Профессия-агностик:** менее ценно для невизуальных профессий (юрист, IT). Для всех визуальных (бьюти, груминг, татту, фото, дизайн интерьера, ремонт, кулинария) — да.
**Priority: EXPERIMENTAL. Complexity: M. Tier: PRO.**

## 12. Tax-document generator (Налоговый отчёт)

**Триггер:** "Сформировать отчёт самозанятого" в финансах.
**Входные данные:** все доходы за период, ИНН мастера.
**Выход:** PDF с разбивкой по дням, итогами, расчётом налога 4%/6%, готово для приложения "Мой налог" или ФНС.
**UX:** один тап → PDF в Share Sheet.
**Реализация:** **on-device** генерация PDF. Никакого AI здесь не нужно — это шаблон.
**Стоимость:** **0 ₽**.
**Профессия-агностик:** **CIS-specific**, но критично для позиционирования. Для глобальной экспансии — аналог под другие юрисдикции (UK Sole Trader, US Schedule C).
**Priority: MUST для CIS. Complexity: M. Tier: PRO.**

## 13. Pricing optimizer

**Триггер:** раз в квартал в Inbox.
**Входные данные:** услуги мастера и цены; **проблема — competitor data**.
**Реалистичный подход:** не парсить конкурентов (юридически и технически сложно), а сравнивать **с обезличенным пулом MasterBook-мастеров той же профессии в том же регионе**. Это уникальный data moat — никто кроме нас не сможет такое предложить.
**Выход:** "Мастера маникюра в Казани в среднем берут 1800 ₽ за классику. Ты — 1500. Подумай о повышении." Анонимно, агрегировано (минимум 10 мастеров в выборке).
**UX:** карточка в Inbox раз в квартал.
**Реализация:** серверная агрегация (Supabase RPC) + on-device рендеринг.
**Стоимость:** ~**0.5 ₽/мес** инфраструктуры.
**Профессия-агностик:** да, и это создаёт **network effect** — чем больше мастеров одной профессии, тем точнее данные.
**Priority: NICE (требует критической массы). Complexity: L. Tier: ELITE.**

## 14. Client LTV score

**Триггер:** фоновый.
**Входные данные:** spend history, visit frequency, retention, referrals.
**Выход:** predicted LTV за 12 месяцев на клиента.
**UX:** виден в карточке клиента (₽ badge), фильтр "топ-20% клиентов" — кому слать персональные акции.
**Реализация:** **on-device**, простая формула (avg_check × predicted_visits_per_year × retention_probability). Без ML.
**Стоимость:** **0 ₽**.
**Priority: NICE. Complexity: S. Tier: PRO.**

## 15. Anti-burnout flag

**Триггер:** воскресенье вечером, при просмотре расписания на след. неделю.
**Входные данные:** часы записей за неделю, кол-во дней без выходного, кол-во "коротких слотов".
**Выход:** "На следующей неделе 58ч записей и нет выходного. Заблокировать слот?"
**UX:** мягкая карточка в Inbox, не модальное окно. Никогда не блокирует.
**Реализация:** **on-device**, пороги настраиваются мастером.
**Стоимость:** **0 ₽**.
**Профессия-агностик:** да, **выраженно ценно** для самозанятых везде. Это про заботу — формирует эмоциональную связь с продуктом.
**Priority: NICE. Complexity: S. Tier: Free (отличие от конкурентов).**

## Дизайн AI-Assistant UI/UX

### Где живёт

**Не отдельная вкладка**. AI растворён в продукте, не торчит как маркетинговая нашлёпка.

**Три поверхности:**

1. **Inbox-экран** (новая вкладка между "Главная" и "Клиенты") — единая лента AI-инсайтов, как уведомления, но не push'и. Здесь живут: idle-slot suggestions, churn alerts, anti-burnout, pricing tips, weekly digest. Максимум 3-5 активных карточек одновременно — остальные сворачиваются в архив. Карточка свайпится "отклонить" / "напомнить позже" / "действовать".

2. **Inline-подсказки** в существующих экранах. No-show badge на appointment-карточке. Duration-suggestion под полем длительности. LTV-badge на карточке клиента. Master вообще может не думать что это "AI" — это просто умное приложение.

3. **AI-кнопка (sparkles ✨)** в правом верхнем углу — призывная on-demand форма. Тап → bottom-sheet с командами: "заполнить окна", "сделать прогноз", "найти спящих", голосовой ввод "запиши Аню на завтра". Это для опытных юзеров.

### Когда AI проактивен vs пассивен

**Правило золотого молчания:** AI говорит когда:
- есть **конкретный actionable item** (не "вот тебе аналитика", а "сделай X"),
- ожидаемая ценность ≥ 500 ₽ потенциального дохода или ≥ 30 минут экономии,
- не чаще **1 уведомления в день** (digest format).

**Никогда:** push-уведомлений с "интересными фактами". Только при открытии app.

**Раз в неделю (воскресенье 19:00):** "Сводка недели" — заработал X, провёл Y часов, на след. неделе Z записей, есть N свободных слотов. Это единственный регулярный proactive push, и его можно выключить.

### Тональность

Русский, на "ты", тёплый но не сюсюкающий. Без "🌸✨💖". Эмодзи только функциональные (⚠️ 💡 ✅). Без слова "искусственный интеллект" в UI — это про продукт, а не про тех. Брендинг: "Ассистент" или просто умные подсказки без названия.

**Хорошо:** "Лена не была 11 недель. Написать?"
**Плохо:** "🤖 Наш ИИ заметил, что Лена давно не была! Хотите воспользоваться AI-генерацией сообщения? ✨"

### Анти-раздражение

- **Snooze everything**: любой инсайт → "не сейчас" / "не показывай больше".
- **Why am I seeing this?**: тап на любую AI-карточку → объяснение факторов. Прозрачность = доверие.
- **Opt-in для кросс-данных**: pricing optimizer и любой server-side feature требует явного согласия в onboarding ("Поделиться обезличенными данными для сравнения с другими мастерами? Это поможет тебе видеть конкурентные цены").
- **Privacy-first**: всё что можно — on-device. В onboarding явно: "AI-ассистент работает прямо на твоём телефоне. В облако уходит только то, для чего нужно (OCR чека, голос)".

## Что это даёт стратегически

**Free tier** (#3, #6 базовый, #9, #15) — достаточно умный чтобы зацепить, но без killer-features.
**PRO 299 ₽/мес** — оправдан #1, #2, #7, #8, #12 (только idle-slot suggestions окупает подписку за один заполненный слот).
**ELITE 599 ₽/мес** (новый tier) — #10 голос, #13 pricing intelligence, безлимит OCR, приоритетная поддержка. Для мастеров с оборотом 100k+ ₽/мес.

**Profession-agnostic делается через:**
- В onboarding выбор профессии → набор пресетов услуг и шаблонов сообщений меняется,
- AI-фичи #1-#15 универсальны — все опираются на абстракции "клиент / запись / услуга / цена", не на "маникюр".
- Маркетинговые материалы по сегментам (лендинг для репетиторов, для барберов, для фотографов) с теми же фичами под их жаргон.

**Network effect** через #13 — единственная фича которая становится лучше с ростом пользовательской базы и которую невозможно скопировать.
