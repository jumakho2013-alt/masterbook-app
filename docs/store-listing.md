## Google Play — Store listing (RU)
**Название (≤30):** MasterBook
**Краткое описание (≤80):**
CRM для частных мастеров: клиенты, записи и доходы — в одном приложении.

**Полное описание (≤4000):**
MasterBook — простое приложение для частных мастеров: маникюр, барбер, брови, ресницы, косметология, массаж, репетиторы и не только.

• Записи и расписание — добавь клиента и запись за несколько касаний, видно занятые слоты и «окна».
• База клиентов — история визитов, заметки, предпочтения, долги, импорт из контактов.
• Финансы — доход, расход, чистая прибыль, средний чек; отчёт о доходах PDF для самозанятых.
• Напоминания — вечером подскажет разослать клиентам напоминания на завтра (WhatsApp/SMS), тексты уже готовы.
• Работает офлайн — данные на телефоне; с аккаунтом ещё и в облаке, не потеряешь при смене телефона.
• Без рекламы, без комиссий с твоих записей.

Русский и английский. Тёмная и светлая темы.

**Что нового (v1.0.0):**
Первый релиз: записи, клиенты, финансы, напоминания, облачная синхронизация, публикация профиля.

## Google Play — Data Safety (анкета, ответы)
- Собираются ли данные? **Да.**
- Типы: **Имя**, **Email** (если регистрируется), **Телефон**, **Фото** (фото работ), **Другой пользовательский контент** (клиенты/записи/заметки, которые вводит мастер).
- Цель: **функциональность приложения** (не реклама, не аналитика третьих сторон).
- Передаётся третьим лицам? **Нет.** Продаётся? **Нет.**
- Шифрование при передаче? **Да** (HTTPS/Supabase).
- Можно ли запросить удаление? **Да** — удаление аккаунта в приложении (Профиль → Безопасность → Удалить аккаунт).
- Реклама / трекинг? **Нет.**

## App Store — App Privacy (nutrition label)
- Data Linked to You: **Contact Info** (имя, email, телефон), **Photos**, **User Content**.
- Used for tracking? **No** (в app.json `NSPrivacyTracking=false`).
- Цель: App Functionality.

## Что осталось ТОЛЬКО ТЕБЕ (нужен твой вход/ключи)
1. **iOS-сборка:** `eas build -p ios --profile production` в терминале → войти в Apple ID + 2FA → «Generate certificate?» Yes.
2. **Android → Play:** скачать `.aab` со страницы билда и загрузить в Play Console, **или** положить `google-service-account.json` в корень и `eas submit -p android --profile production --latest`.
3. **iOS → ASC:** `eas submit -p ios --profile production --latest` → TestFlight → Submit for Review.
4. **В консолях:** вставить тексты выше, ответить на Data Safety / App Privacy (ответы выше), указать Privacy Policy URL, приложить скриншоты (5–8 шт. с реальных экранов), нажать «Отправить на ревью» / rollout.
