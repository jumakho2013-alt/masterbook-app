# MasterBook* — Мастер-план: продукт + сайт + платежи + шаги

> Единый источник правды. *«MasterBook» — рабочее кодовое имя; название занято, поменяем перед публичным запуском (это косметика: `app.json` name/slug + i18n + лого + store-listing, меняется за минуты, на код/данные не влияет).
> Дата: июнь 2026. ЦА: СНГ — Таджикистан (домашний рынок), Узбекистан, Казахстан, Кыргызстан, Россия.

## 0. Идея в одной фразе
Бесплатное приложение-CRM для мастеров (магнит) → сайт-каталог, где клиенты их находят и записываются → мастер платит **за продвижение в каталоге**, оплата локальным переводом/QR мимо сторов. Деньги клиента мы НЕ трогаем. Цель — стабильный плюс, не миллионы (< 10 платящих мастеров покрывают расходы).

## 1. Архитектура — один организм
- **Приложение мастера (готово, React Native):** ежедневный инструмент — записи, клиенты, финансы, портфолио. Это «приложение исполнителя» по стандарту индустрии.
- **Сайт (Next.js):** витрина-каталог для клиентов + кабинет мастера + админка для тебя.
- **Общая база Supabase** (auth + БД + storage): профиль/услуги/часы/портфолио — одни и те же. Заявки с сайта → падают мастеру в приложение.
- Принцип: телефон = инструмент мастера; сайт = витрина для клиентов; одна база их связывает.
- Аналог-шаблон 1:1 — **DIKIDI** (DIKIDI Business + DIKIDI Online + сайт; freemium + платное продвижение, не комиссия с брони).

## 2. Регистрация / вход (максимально легко)
- **Клиент — БЕЗ регистрации.** Запись = имя + телефон + SMS-код. Без паролей.
- **Мастер — тот же аккаунт, что в приложении.** Вход: телефон + SMS (основной, универсален в СНГ), Google/Apple — опционально.
- Всё на **Supabase Auth** (phone OTP + OAuth), отдельно ничего не строим.

## 3. Сайт — структура (три входа)
### Клиент (открытая часть)
- **Главная:** поиск (услуга + город), категории-плитки с иконками, блок «лучшие мастера города».
- **Каталог:** фильтры (категория / район / цена / рейтинг / «свободен сегодня»); карточки мастера (фото-работа или монограмма, имя, услуга, район, цена «от» серифом, звёзды); premium-мастера — вверху с золотой меткой.
- **Профиль мастера (ключевой экран):** шапка (обложка работ, имя серифом, рейтинг золотыми звёздами, VIP-метка золотом); разделы «Услуги и цены» (цены серифом), «Мои работы» (портфолио-галерея — для бьюти главное), «Отзывы», «О мастере», «Где находится» (район/карта); липкая кнопка «Записаться» (плам) + WhatsApp + Позвонить.
- **Онлайн-запись:** услуга → дата + свободный слот → имя + телефон → SMS-код → «Готово, вы записаны».
### Мастер (кабинет = как в приложении, синхронно)
- **Профиль:** фото, услуги+цены, рабочие часы, район, контакты, загрузка «Мои работы» (портфолио). Правит в телефоне или на сайте — одна база, синхронно.
- **Заявки:** входящие с сайта (и в приложении).
- **Продвижение:** пакеты «топ в каталоге» → оплата → premium.
- **Баланс / статус.**
### Ты (админка — пульт)
- Мастера, балансы, **неподтверждённые оплаты → «подтвердить» (1 клик)**, кто premium, статистика.
### Профиль «под тип мастера»
Один гибкий шаблон, контент по категории (уже есть профессия-паки): бьюти — портфолио вперёд; репетитор/уборка — «о мастере»/отзывы.

## 4. Монетизация и ПЛАТЕЖИ (главное — из исследования по 5 странам)
### Модель
Платит ТОЛЬКО мастер, за **продвижение** (топ каталога) — копия DIKIDI Premium (+30% записей у них). Freemium. **НЕ комиссия с брони** (YouDo ушёл от неё — обходят; TaskRabbit-модель требует эквайринг/эскроу — избегаем).

### Ключевая абстракция — ВНУТРЕННИЙ БАЛАНС мастера
Мастер пополняет баланс **крупным платежом** (месяц/квартал), продвижение **списывается с баланса** (мгновенно, бесплатно). Это спасает экономику микро-платежей ($0.5–5): поштучные платежи убивают комиссии и ручная сверка.

### MVP приёма (без юрлица, для ВСЕХ 5 стран): ручной перевод + подтверждение
1. Мастер в приложении **выбирает свою страну** → видит реквизиты **именно своей страны** (локальный P2P — мгновенно/дёшево). НЕ заставляем платить «в Таджикистан».
2. **Полу-авто сверка:** уникальная сумма-маркер (5.07 вместо 5.00) или код-комментарий = ID мастера → matching почти автоматический даже без webhook. Где можно — парсить SMS/пуш банка.
3. Мастер жмёт «Я оплатил» → подтверждение (вручную/скриптом) → активация.
4. **Идемпотентность ОБЯЗАТЕЛЬНА:** ключ = (ID мастера + сумма + референс/окно времени). Двойной тап / двойной перевод / refresh / повтор webhook — НЕ задваивают активацию и НЕ списывают баланс дважды.

### Реквизиты по странам
- **TJ (дом, старт отсюда):** ручной — Korti Milli / Alif Mobi / DC Next + национальный **единый QR**. Авто (этап 2, после ИП): **Smartpay.tj** — payment-link + QR + API/webhook поверх ВСЕХ местных кошельков и карт, **фикс-комиссия 0,5–1 сомони** (идеал для мелочи).
- **RU:** ручной — перевод на твою TJ-карту (трансгран. СБП / Юнистрим / DC Next / МТС). Авто (этап 2, без юрлица): **идентифицированный кошелёк ЮMoney** (идентификация офлайн в «Душанбе Сити») → кнопка/QR + webhook (с проверкой подписи!). **ЮKassa гражданину TJ закрыт.**
- **UZ:** ручной — Uzcard/Humo по номеру. Merchant-API (Payme/Click/Uzum) требует узб. бизнес-счёт → **тупик для нерезидента**; авто только через местного партнёра/ИП (этап 3).
- **KZ:** ручной — Kaspi Gold по номеру. ⚠️ **с 01.01.2026 фискальный мониторинг P2P** — не строить на регулярном P2P на личную карту; авто через ИП KZ (Halyk ePay / Kaspi Business) — этап 3.
- **KG:** ручной — MBank + национальный **ELQR-QR**. ⚠️ **с 01.01.2026 запрет личного оборудования в коммерции**; авто через ИП (MKassa / O!Business) — этап 3.

### Cross-border — честно
Единого международного приёма для TJ-физлица **НЕТ**. Собирать **локально по странам**, не «всё в TJ». Порядок авто: TJ (Smartpay+ИП) → RU (ЮMoney) → UZ/KZ/KG позже и локально.

## 7. Риски

- **Микро-платёж-экономика:** только пакеты/баланс, не поштучно.
- **Идемпотентность + проверка подписи webhook (anti-spoofing)** — обязательны.

- **НЕ использовать:** USDT (серая зона TJ, НБТ враждебен), Payoneer/Wise (не открывают TJ-резиденту / дорого на микро-суммах).
