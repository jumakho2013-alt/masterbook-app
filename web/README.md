# MasterBook — сайт-каталог (Next.js)

Витрина мастеров поверх **той же Supabase**, что и мобильное приложение. Читает
**публичный слой** (`published`-мастера + их услуги + отзывы) — приватная CRM
(клиенты/записи/финансы) анониму недоступна (RLS).

## Что готово (MVP, читает живую базу)
- **Главная** — hero, категории, лучшие мастера.
- **Каталог** (`/catalog`) — список опубликованных мастеров, поиск, премиум-первыми.
- **Профиль мастера** (`/m/<slug>`) — услуги+цены (сомони), отзывы, рейтинг, VIP,
  кнопка **«Записаться»** → WhatsApp/звонок (поле задаётся в приложении: Профиль →
  «Моя страница на сайте»).

## Запуск
```bash
cd web
cp .env.local.example .env.local   # вставь те же значения, что в app .env.local
#   EXPO_PUBLIC_SUPABASE_URL      → NEXT_PUBLIC_SUPABASE_URL
#   EXPO_PUBLIC_SUPABASE_ANON_KEY → NEXT_PUBLIC_SUPABASE_ANON_KEY
npm install
npm run dev        # http://localhost:3000
```
Чтобы в каталоге кто-то появился: в приложении у мастера зайди Профиль → «Моя
страница на сайте» → заполни город + укажи услуги → включи «Показывать на сайте».

## Чего ещё НЕТ (следующие итерации)
- **Онлайн-запись в аккаунт мастера** — нужен Supabase Edge Function (анонимная
  запись с телефоном+SMS пишет в `appointments` под service-role; прямой anon-insert
  закрыт RLS специально). Пока «Записаться» = WhatsApp/звонок.
- **Кабинет мастера** на сайте (вход тем же Supabase Auth) и **админка** (подтверждение
  оплат `confirm_payment`, премиум).
- **Оплата/баланс** на сайте (таблицы `payments`/`master_billing`/`promotions` и функции
  уже есть в БД) — UI + реальные реквизиты по странам (твоя рука).
- **Портфолио-картинки**: бакет Storage сейчас приватный — для публичных фото нужен
  публичный бакет/политика.
- Тёмная тема (Noir) из прототипа, мультигород, фильтр по району.
